# 从一张图做出自己的 Codex 皮肤（Windows）

这份指南解决的是完整流程，而不是提供某一套成品主题：

> 你准备一张自己的背景图，仓库负责把安装、换图、验证、排错和恢复串起来。

主题可以使用你有权再分发的原创插画、风景、宠物照片、品牌视觉或抽象图形；制作前先确认素材权利与交付边界。

如果你准备把仓库直接交给 AI，并用“学习这个仓库，打造一个某某皮肤”开始，请先读[尽量一次成功：自定义 Codex 皮肤工作流](./one-shot-skin-workflow.md)。它定义了通用的需求分类、默认值、Agent 执行顺序、提示词模板和验收矩阵。

## 你最后会得到什么

- Codex 首页使用你的图片和自动提取的主题色。
- 侧栏、项目选择器和输入框仍是 Codex 原生控件，可以正常操作。
- 普通任务页会降低图片干扰，不会把聊天内容盖住。
- 系统托盘可以换图、保存主题、切换主题、暂停皮肤或完全恢复 Codex。
- 安装失败时会保留错误弹窗和日志，不再只看到终端闪一下。

## 1. 准备一张合格的背景图

推荐使用 `2560 × 1440`、接近 16:9 的 PNG 或高质量 JPEG。它应该是一张**纯背景图**：

- 不要画入侧栏、输入框、按钮、卡片、标题或 Codex 标志。
- 不要把提示词、签名、水印或装饰符号放在界面文字可能出现的区域。
- 主体建议放在画面右侧约 `58%–88%` 的区域，左侧保留低信息安全区。
- 主体的眼睛和面部不要贴近窗口边缘，避免窗口缩放后被裁掉。
- 背景需要有足够明暗层次，但不要在整张图上铺高对比碎片。

需要用 AI 生图时，先阅读[参考图生图提示词](./reference-background-prompt-guide.md)。核心要求是：**只生成视觉素材，不生成任何界面**。

## 2. 安装皮肤工具

开始前确认：

- 你已从 Microsoft Store 安装并登录官方 Codex Windows 应用。
- 电脑已安装 Node.js 22 或更高版本。
- 仓库已经完整解压，不要直接在压缩包预览窗口里运行。

然后：

1. 关闭正在运行的 Codex，或在安装弹窗出现时同意关闭。
2. 双击 [`windows/Install Codex Dream Skin.cmd`](../windows/Install%20Codex%20Dream%20Skin.cmd)。
3. 等待成功提示。安装器会启动带皮肤的 Codex，并创建 Launch、Tray、Restore 快捷方式。
4. 如果桌面只出现一个快捷方式，到开始菜单的 **Codex Dream Skin** 文件夹寻找另外两个；具体位置会随 Windows 桌面重定向设置变化。

以后不要点普通 Codex 图标来判断皮肤是否存在。使用 **Codex Dream Skin - Launch** 启动；托盘程序负责换图和切换主题。

## 3. 换成你自己的图片

1. 找到任务栏右下角的 Codex Dream Skin 托盘图标；看不到时先展开隐藏图标。
2. 右键选择 **更换背景图**。
3. 选择第 1 步准备好的 PNG 或 JPEG。
4. 回到 Codex 新建任务页查看效果。必要时选择 **应用或重新应用**。
5. 满意后选择 **保存当前主题**，输入一个容易辨认的名字。

导入图片时会复制到 `%LOCALAPPDATA%\CodexDreamSkin` 的受管目录。原图移动或删除后，已保存主题仍能继续使用。

## 4. 先做这五项验收

| 检查位置 | 合格标准 |
|---|---|
| 新建任务页 | 图片主体完整，标题和项目选择器清楚可读 |
| 输入框 | 没有图片自带的按钮、图标或“脏东西”混入输入区域 |
| 普通任务页 | 背景不会穿过聊天内容，也没有漂浮装饰跨越多个区域 |
| 侧栏 | 文字、选中状态和滚动条有足够对比度 |
| 恢复流程 | **Codex Dream Skin - Restore** 可以关闭皮肤并恢复官方启动方式 |

建议分别测试最大化窗口、较窄窗口、浅色外观和暗色外观。皮肤好看只是第一关，文字可读和随时可恢复同样重要。

## 5. 我们实际踩过的坑

| 现象 | 原因或判断 | 处理办法 |
|---|---|---|
| 双击后终端闪一下，什么也没发生 | 旧入口关闭太快，真实错误被吞掉 | 使用本 Fork 的 `.cmd` 引导入口；查看 `%LOCALAPPDATA%\CodexDreamSkin\*-error.log` |
| 安装提示 `Refusing to rewrite nested desktop tables` | `config.toml` 中存在嵌套或重复的 `[desktop.*]` 表 | 当前安装器会先规范化为单一 `[desktop]`；不要手工删除其他项目配置 |
| 安装似乎成功，随后提示“拒绝访问” | Microsoft Store 启动链可能丢失调试参数，或限制直接启动 | 保持默认 `Auto`；它会先尝试官方启动，失败后使用本机私有运行副本 |
| 只看到界面变绿，没有看到图片 | 启动的是普通 Codex，或皮肤尚未重新应用 | 从 **Codex Dream Skin - Launch** 启动，并在托盘点 **应用或重新应用** |
| 图片只显示成横幅，不是沉浸全幅 | 超宽图片或 `taskMode` 被识别为 `banner` | 使用 16:9 背景；进阶用户可参考[平台能力](./platforms.md)调整 `taskMode` |
| 装饰小点跨过聊天窗口显示 | 装饰被注入成全局浮层，而不是图片的一部分 | 把装饰画进背景图并限制在安全区；不要给装饰层使用跨窗口的全局定位 |
| 输入框里出现莫名的小图标或脏点 | 图片带了假 UI，或注入样式错误命中了输入区 | 换用纯背景图；先暂停皮肤确认来源，再重新应用或反馈 Issue |
| 想彻底回到官方界面 | 只关托盘不能清除当前调试会话 | 双击 **Codex Dream Skin - Restore**，或在托盘选择 **完全恢复 Codex** |

### 关于本机约 1.8GB 的运行副本

它不是皮肤内容，也不会上传到 GitHub。

当 Microsoft Store 的启动方式丢失必要参数时，`Auto` 模式会把你电脑上已经验证过的官方 Codex 运行文件复制到仓库内被忽略的 `windows/runtime/`，仅用于本机启动。首次复制大约占用 1.8GB；Codex 更新后会自动刷新。仓库的 `.gitignore` 和发行检查会阻止这些文件进入提交或 Release。

如果你不接受这份本地磁盘占用，可以使用 Restore 恢复官方启动，并删除本机 `windows/runtime/`；代价是在部分 Store 环境里皮肤可能无法启动。

## 6. 怎样反馈一个有用的 Bug

提交 Issue 时请附上：

- Windows 版本与 Codex 版本。
- 你双击的入口名称。
- 错误弹窗的完整文字。
- `%LOCALAPPDATA%\CodexDreamSkin\*-error.log` 中与本次失败相关的内容；先删除用户名、路径或其他不想公开的信息。
- 一张能看清问题位置的截图；发布前遮住任务名称、账号和聊天内容。
- 问题发生在新建任务页、普通任务页，还是两者都有。

不要上传 `windows/runtime/`、官方 Codex 程序文件、登录信息或完整个人配置。
